# วิธีอัปเดตเกมแบบง่ายที่สุด

คู่มือนี้ใช้หลังจาก GitHub, Render, Cloudflare Pages และ Neon เชื่อมกันเรียบร้อยแล้ว

## ทุกครั้งที่ได้รับ ZIP รุ่นใหม่

1. ดาวน์โหลด ZIP
2. คลิกขวา ZIP แล้วเลือก **Extract All / แยกไฟล์ทั้งหมด**
3. เปิดโฟลเดอร์ `legends-game` ที่อยู่ข้างใน
4. เปิด GitHub Repository `legends-game` ผ่านเว็บ
5. กด **Add file** → **Upload files**
6. ลาก **ทุกไฟล์และทุกโฟลเดอร์ที่อยู่ข้างใน `legends-game`** ลงช่องอัปโหลด
7. รอจนรายการไฟล์โหลดครบ
8. ช่อง Commit message ใส่ชื่อรุ่น เช่น `Update game to M2.6`
9. เลือก **Commit directly to the main branch**
10. กด **Commit changes**
11. รอ Render และ Cloudflare Pages Deploy อัตโนมัติ
12. เปิดลิงก์ `.pages.dev` เพื่อเล่น

## อัปโหลดทั้งหมดได้ไหม

ได้ — ZIP รุ่นนี้ถูกจัดให้มีเฉพาะไฟล์ที่ควรขึ้น GitHub แล้ว

ใน ZIP ไม่มี:

- `.env`
- Secret หรือรหัสผ่าน
- `node_modules`
- `dist`
- Database URL
- JWT Secret
- Log จากเครื่องพัฒนา

ไฟล์ `.env.example` อัปโหลดได้ เพราะเป็นเพียงรายชื่อค่าที่ต้องตั้ง ไม่มีรหัสจริง

## สิ่งที่ไม่ต้องทำอีก

- ไม่ต้องรัน Migration ด้วยตนเอง
- ไม่ต้องรัน Seed ด้วยตนเอง
- ไม่ต้องใช้ PowerShell
- ไม่ต้องติดตั้ง Node.js เพื่อ Deploy
- ไม่ต้องกด Deploy ที่ Render/Cloudflare ในการอัปเดตปกติ

## กรณีอัปโหลดไฟล์ทับผ่าน GitHub Web

GitHub Web อาจไม่ลบไฟล์เก่าที่ไม่มีอยู่ใน ZIP ใหม่โดยอัตโนมัติ ดังนั้นโปรเจกต์ออกแบบให้หลีกเลี่ยงการเปลี่ยนชื่อหรือลบไฟล์โดยไม่จำเป็น หากรุ่นใดจำเป็นต้องลบไฟล์ จะระบุชื่อไฟล์ที่ต้องลบให้ชัดเจนเป็นกรณีพิเศษ
